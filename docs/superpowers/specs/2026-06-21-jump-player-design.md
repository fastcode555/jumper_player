# Jump Player 设计文档

> 跨平台本地视频播放器，支持智能跳过片头片尾、截图、做 GIF。
> 日期：2026-06-21
> 状态：设计已确认，待写实现计划

## 1. 目标与范围

### 1.1 产品目标

一个 **write once, run anywhere** 的本地视频播放器，单一代码库覆盖 **macOS / Linux / Windows / Android / iOS** 五端。核心差异化能力：

- 强格式兼容的本地播放（libmpv 级别）
- 按"剧集组"统一设定**跳过片头片尾**，标记一集、全组生效
- **截图**与**做 GIF**
- 无边框窗口 + 沉浸式全屏 + 闲置自动隐藏光标/控制栏

### 1.2 第一期范围（YAGNI）

**做**：

- 本地播放（全格式、硬解、字幕、倍速、音轨切换、续播）
- 媒体库：添加根文件夹 → 递归扫描 → 集号解析 → 有序剧集列表
- 上一集/下一集（跨子文件夹无感）、播完自动下一集
- 手动片头片尾区间，按剧集组套用，自动跳或显示按钮
- 截图（PNG/JPG）
- 做 GIF（区间 + 帧率/分辨率可调）
- 窗口/沉浸式 UI：无边框、全屏、闲置隐藏

**不做（预留接口，不实现）**：

- 自动识别片头片尾（B 方案，音频指纹/帧比对）
- 在线源、投屏、转码、字幕在线下载

### 1.3 核心设计原则

- **UI 框架 ≠ 播放引擎**：格式兼容与播放效果取决于底层 libmpv/FFmpeg，不依赖 Flutter 自带播放控件。
- **领域服务层与 Flutter/平台完全解耦**，纯 Dart、可单测，是整套设计可被信任的关键。
- **唯一允许分平台的地方**是文件访问与窗口适配，其余 write once。

## 2. 技术栈

| 关注点 | 选型 | 理由 |
|---|---|---|
| UI 框架 | Flutter（全 5 端单一代码库） | 真五端、迭代快、生态成熟 |
| 播放引擎 | media_kit（libmpv 封装） | mpv 级格式兼容、硬解、字幕、HDR |
| 截图/GIF | media_kit/mpv screenshot + FFmpeg | 精确帧截图 + 高质量 GIF |
| 状态管理 | Riverpod | 可注入 mock、测试友好、领域层不依赖 Flutter |
| 本地存储 | Drift（SQLite） | 类型安全、可迁移、可单测 |
| 窗口管理 | window_manager | 桌面无边框、全屏、红绿灯适配 |

## 4. 媒体库与扫描

### 4.1 剧集组模型

**剧集组（Series）的边界 = 用户添加进库的那个根文件夹**，而非视频的直接父文件夹。对根目录**递归遍历**，把所有视频文件拉平成一个有序的剧集列表。两种常见布局自动统一：

```
情况1：一集一文件夹（嵌套）          情况2：扁平
权力的游戏/                          权力的游戏/
├─ E01/ 权游.S01E01.mkv             ├─ 权游.S01E01.mkv
├─ E02/ 权游.S01E02.mkv             ├─ 权游.S01E02.mkv
└─ E03/ 权游.S01E03.mkv             └─ ...
```

两者递归扫描后得到同一有序列表 `[E01, E02, E03...]`；上一集/下一集即在该列表索引 ±1，**不关心文件嵌套深度**。

### 4.2 扫描流程（LibraryScanner）

- 在后台 **isolate** 异步递归遍历根目录，避免阻塞 UI。
- 仅收集视频扩展名：`mkv/mp4/avi/mov/flv/ts/webm/m4v/wmv/rmvb/...`，跳过字幕/nfo/图片。
- 每个文件经 EpisodeSorter 提取 (季, 集) 并排序。
- **增量缓存**：库表记录 `path + mtime + size + 时长 + 集号`；再次进库仅扫新增/变动文件，不全量重扫。
- 文件不存在/被移动 → 标记失效，不崩溃，提示重新定位。

### 4.3 集号提取（EpisodeSorter）

按优先级正则匹配，命中即停：

| 优先级 | 模式 | 示例 | 提取 |
|---|---|---|---|
| 1 | `SxxExx` | `权游.S01E02.mkv` | 季1 集2 |
| 2 | 中文"第N集/话/期" | `吞噬星空第123集` | 集123 |
| 3 | 中文"N集"（无"第"） | `吞噬星空123集` | 集123 |
| 4 | `EP123` / `E123` | `EP123.mp4` | 集123 |
| 5 | 末尾独立数字 | `吞噬星空123.mp4` | 集123 |
| 6 | 全部失败 | `片头曲.mkv` | 无集号 → 路径自然排序 |

**自然排序**：`9集` 排在 `123集` 之前（按数值，非字典序）。

**已知坑及对策**：

- 数字干扰项（`吞噬星空4K.123集.mp4` 中的 `4`）：靠"第N集/集"高优先级规则避开，不抓第一个数字。
- 年份/分辨率（`某剧.2024.1080p.E05.mkv` 中的 `2024`/`1080`）：`SxxExx`、"第N集"优先级最高先命中，不误判。
- 同集多文件（视频 + `.ass` 字幕）：非视频扩展名跳过。
- 提取规则**可配置/可扩展**：新命名习惯加一条正则即可，不动主逻辑。

### 4.4 手动修正（D 方案）

- 把一个根目录手动**拆**成多组（混放多剧/多季）。
- 把多个根目录**合并**为一组。
- 手动修改单个文件的集号。

## 5. 播放引擎

- media_kit 的 `Player` + `VideoController`（libmpv 后端）：全格式、硬解、内封/外挂字幕、倍速、音轨切换。
- 上一集/下一集 = 剧集列表索引 ±1，跨子文件夹无感。
- **续播**：记忆每集播放进度，下次从断点继续。
- **自动下一集**：播完自动续播下一集（可在设置关闭）。

## 6. 片头片尾跳过（SkipRuleEngine）

### 6.1 数据模型

规则挂在 Series 上（不是单个文件）：

```
SkipRule {
  seriesId
  introStart, introEnd     // 片头区间（秒）
  outroStart               // 片尾起点（秒），到结尾
  mode: 'manual'           // 预留 'auto' (B 方案)
}
```

### 6.2 交互与行为

- 用户在任意一集点"标记片头结束 / 标记片尾开始" → 写入该 Series 规则 → **全组生效**。
- 播放时 SkipRuleEngine 监听进度：
  - 进入片头区间 → 自动 seek 到 `introEnd`，**或**显示"跳过片头"按钮（二选一，设置项）。
  - 到达 `outroStart` → 跳下一集。

### 6.3 B 方案预留

- `mode` 字段 + `SkipDetector` 接口占位；将来塞音频指纹/帧比对实现，**不改调用方**。

## 7. 截图 / GIF（MediaToolService）

### 7.1 截图

- 优先用 media_kit/mpv 的 `screenshot` 取当前精确帧（可选是否含渲染后字幕）。
- 输出 PNG/JPG 到可配置目录。

### 7.2 GIF

- FFmpeg 生成，用户设：起点 + 时长（默认 3s）+ 分辨率/帧率。
- **两遍法**（先 `palettegen` 生成调色板，再合成）保证质量。
- 进度回调 + 可取消。
- 移动端产物写入 App 可访问目录，调用系统分享。

## 8. 窗口与沉浸式 UI

| 需求 | 实现 | 适用端 |
|---|---|---|
| 无边框窗口 | `window_manager`（`titleBarStyle: hidden`），自绘顶栏 + 拖拽区 + 窗口按钮 | 桌面三端 |
| 全屏 | 桌面 `window_manager.setFullScreen`；移动端 `SystemChrome` 沉浸式 | 全 5 端 |
| 闲置隐藏光标 + 控制栏 | `MouseRegion` 监听移动重置计时器；闲置 **3s（可设置）** → 光标 `SystemMouseCursors.none` + 淡出控制栏；任意移动/按键立即恢复 | 桌面 |

### 8.1 ImmersiveController（统一显隐状态机）

- 统一管理"光标 + 控制栏 + 顶栏"三者一起淡入淡出，避免不同步。
- 输入事件：鼠标移动 / 闲置超时 / 全屏切换 / 按键。
- 可单测：给定事件序列，断言可见性状态。

### 8.2 默认行为

- 闲置触发时间默认 **3 秒**，可设置。
- **仅全屏时**自动隐藏光标；窗口模式只淡出控制栏，顶栏常驻（可设置）。
- 全屏 ↔ 窗口切换时，无边框自绘顶栏在全屏下隐藏、退出全屏恢复，与闲置隐藏共用同一显隐逻辑。

### 8.3 平台适配（WindowAdapter 抽象）

- 无边框/红绿灯是桌面专属；移动端为空实现。仍 write once，分支只在窗口层。
- **macOS 特有细节**：
  - 用 `titleBarStyle: hidden` **隐藏标题栏但保留红绿灯**；自绘顶栏给左上角让位，不压住红绿灯。
  - 调整红绿灯 inset，使其与自绘顶栏高度对齐。
  - 走**自定义沉浸全屏**（覆盖全屏、隐藏菜单栏/Dock/顶栏），而非 mac 原生换 Space 的全屏（参考 IINA 思路）。
  - 闲置隐藏光标时，全屏沉浸下红绿灯一起淡出。

## 9. 数据存储（Drift / SQLite）

表：

- `series`：剧集组（根路径、名称、封面等）
- `episodes`：剧集文件（path、季、集、时长、mtime、size）
- `skip_rules`：片头片尾规则（见 6.1）
- `watch_progress`：每集续播进度
- `settings`：全局设置（闲置时长、自动下一集、跳过模式等）

## 10. 错误处理

- 文件丢失/移动 → 标记失效，不崩溃，提示重新定位。
- 解码失败/缺编解码 → 友好提示 + 降级软解。
- 扫描中断/权限不足 → 跳过并汇总报告。
- 跨端文件权限（Android SAF、iOS 沙盒书签、桌面路径）抽象为 `FileAccess` 接口分端实现。

## 11. 测试策略

- **领域层纯 Dart 单测（重点）**：
  - 集号解析各种命名（`吞噬星空第123集` / `S01E02` / 干扰项 / 年份分辨率 / 失败退回）
  - 自然排序正确性
  - SkipRuleEngine 套用与触发逻辑
  - 扫描归类（嵌套 vs 扁平得到同一列表）
- **ImmersiveController 状态机单测**：事件序列 → 可见性断言。
- **Widget 测试**：播放器交互；media_kit/FFmpeg 用 mock 接口，不依赖真实文件。

## 12. 平台适配清单（唯一分平台处）

| 模块 | 桌面（mac/win/linux） | 移动（android/ios） |
|---|---|---|
| FileAccess | 直接路径 | Android SAF / iOS 沙盒书签 |
| WindowAdapter | 无边框 + 红绿灯(mac) + 沉浸全屏 | 空实现 / SystemChrome 沉浸 |
| 闲置隐藏光标 | 支持 | 仅隐藏控制栏 |
| GIF/截图产物 | 写文件目录 | App 目录 + 系统分享 |

其余全部 write once。
